

La mejor manera de colaborar con la traducción, es simplemente trayéndose el proyecto completo de github, efectuando las traducciones, y luego hacernos un pedido para incluir esas traducciones.

github es una plataforma que está especialmente pensada para facilitar ese tipo de colaboraciones.


* crear una cuenta en github

Así que lo primero que debe hacer es navegar a https://github.com/signup/free y crear una cuenta gratuita en github

* instalar git y condigurar nuestra cuenta de github

Para instalar git en linux, y configurar su cuenta siga este instructivo: http://help.github.com/linux-set-up-git/

Aquí están las instrucciones para hacerlo en windows: http://help.github.com/win-set-up-git/

* Hacer un fork de nuestro repositorio



Navegamos al repositorio del proyecto playdoces en https://github.com/opensas/playdoces

Seguimos esta guia para hacer un fork http://help.github.com/fork-a-repo/

Hacemos click en fork, y nos copiaremos una copia de todo el repositorio, que quedará disponible en

https://github.com/foreversas/playdoces

lo que debemos hacer ahora es hacer una copia local de este repostorio en nuestra estación de trabajo

para ello ejecutamos la siguientes instruccion


mkdir apps
cd apps

git clone git@github.com:foreversas/playdoces.git

git remote add upstream git@github.com:opensas/playdoces.git

cuando querramos traer los cambios ocurridos en el upstream, deberemos ejecutar

git fetch upstream
git merge upstream/master

El esquema de trabajo que nos proponer github puede parecer confuso al princpio, pero en realidad es bastante simple si uno entiende qué rol cumple cada repositorio.

El repositorio original, que tiene la aplicación que será subida a producción a playdoces.appspot.com es el siguiente

https://github.com/opensas/playdoces, y lo llamaremos upstream

Luego hacemos un fork y creamos una copia del repositorio upstream alojado en github en https://github.com/foreversas/playdoces

Sin embargo, no podemos trabajar directamente sobre el repostirio de github, sino que deberemos copiar (clonar) localmente el repositorio para poder efectuar cambios.

Esto lo hacemo con la orden git clone git@github.com:foreversas/playdoces.git.

De forma tal que tenemos lo siguientes repositorios:

git@github.com:opensas/playdoces.git - repositorio upstream en github

git@github.com:foreversas/playdoces.git - repostorio forkeado en github

/devel/apps/playdoces/.git - repostorio clonado en nuestra máquina local

trabajamos en nuestra máquina local, commiteamos en nuestro repositorio local y hacemo sun push a nuestro repostorio forekado en girhub

luego hacemos un pull request pidiéndole al dueño del repostirio upstream que integre los cambios que le enviamos.
